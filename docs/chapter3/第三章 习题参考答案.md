# 第三章 习题参考答案

> 参考答案仅供参考。第三章的习题覆盖模型原理、工程实践与系统设计，建议把“概念解释”和“可落地方案”放在一起理解。

## 1. 从 Bigram 到 RNN/LSTM 与 Transformer

### 使用迷你语料库计算 `agent works` 的 Bigram 概率

语料库为：

```text
datawhale agent learns
datawhale agent works
```

如果按“句子概率”的标准定义，通常会补上句子开始标记 `<s>`：

```text
<s> datawhale agent learns </s>
<s> datawhale agent works </s>
```

于是：

```text
P(agent works) = P(agent | <s>) x P(works | agent)
```

计数结果：

- `C(<s>, agent) = 0`
- `C(<s>) = 2`
- `C(agent, works) = 1`
- `C(agent) = 2`

所以：

```text
P(agent | <s>) = 0 / 2 = 0
P(works | agent) = 1 / 2 = 0.5
P(agent works) = 0 x 0.5 = 0
```

结论：在未做平滑的 Bigram 模型下，`agent works` 的句子概率是 `0`。

这正好体现了 N-gram 的数据稀疏问题。虽然 `works` 的确可以跟在 `agent` 后面，但训练语料里没有出现过“句首直接接 `agent`”，模型就会判成不可能。

补充说明：

- 如果题目只是想问局部条件概率 `P(works | agent)`，那么答案是 `1/2`。
- 如果按完整句子概率来算，标准答案是 `0`。

### 马尔可夫假设的含义

马尔可夫假设的核心是：一个词出现的概率，只依赖于它前面的有限个词，而不依赖更早的全部历史。

例如：

- Unigram: `P(w_n)`
- Bigram: `P(w_n | w_{n-1})`
- Trigram: `P(w_n | w_{n-2}, w_{n-1})`

本质上，它是在用“有限窗口”近似真实语言中的“全部历史依赖”。

### N-gram 的根本局限

- 数据稀疏：高阶 N-gram 组合数量指数爆炸，大量组合在语料中没出现过。
- 长距离依赖差：固定窗口之外的信息直接丢失。
- 无语义理解：只能统计词串共现，不能理解词义与上下文。
- 存储成本高：高阶 N-gram 需要巨大表结构保存计数。

### RNN/LSTM 和 Transformer 如何克服这些局限

#### RNN/LSTM

RNN 的关键思想是用隐藏状态携带历史信息，而不是只看固定窗口。

- 它理论上可以把任意长度的上下文压缩进隐状态。
- 词嵌入使语义相近的词在向量空间中更接近，增强泛化能力。

但普通 RNN 有梯度消失问题。LSTM 通过门控机制和独立的记忆单元缓解这一问题，使模型能更稳定地保留较长上下文信息。

RNN/LSTM 的优势：

- 比 N-gram 更能建模序列历史。
- 比统计模型更有语义泛化能力。

#### Transformer

Transformer 的核心突破是自注意力。

- 任意两个位置可以直接建立联系。
- 不再依赖串行递推，因此可以并行训练。
- 每个词的表示是动态上下文化的，同一个词在不同上下文中可以有不同表示。

Transformer 的优势：

- 更强的长距离依赖建模能力。
- 训练并行度高，适合大规模扩展。
- 更容易形成强大的上下文理解和生成能力。

## 2. Transformer 的核心机制

### 自注意力机制的核心思想是什么

一句话概括：让序列中的每个位置都能动态评估与其他位置的相关性，并按相关性聚合信息。

它通常通过 `Q/K/V` 三组向量实现：

- Query: 我当前需要什么信息。
- Key: 我能提供什么信息。
- Value: 我的实际内容是什么。

计算流程如下：

```text
Score = Q x K^T / sqrt(d_k)
Attention = softmax(Score)
Output = Attention x V
```

直观理解：一个词在理解自己时，会去“看”句子里哪些词最相关，并把这些词的信息按权重吸收进来。

### 为什么 Transformer 能并行，而 RNN 必须串行

RNN 是递推结构：

```text
h_t = f(h_{t-1}, x_t)
```

所以第 `t` 步必须等第 `t-1` 步完成，天然串行。

Transformer 不依赖前一步隐状态，它对整段输入一次性做矩阵计算，因此所有位置可以并行处理。这是它能高效利用 GPU、大规模训练的关键原因。

### 位置编码的作用

自注意力本身不理解顺序。对它来说，“猫追狗”和“狗追猫”如果只看词集合，区别并不天然存在。

因此需要位置编码，把顺序信息注入模型。常见方式包括：

- 原始 Transformer 的正弦余弦位置编码。
- 可学习位置编码。
- 现代 LLM 常用的 RoPE 旋转位置编码。

### Decoder-Only 和 Encoder-Decoder 的区别

#### Encoder-Decoder

- Encoder 用双向注意力理解输入。
- Decoder 在生成时既看自己已生成内容，也看 Encoder 输出。
- 适合翻译、摘要这类明确的序列到序列任务。

#### Decoder-Only

- 只有单向因果注意力。
- 本质上把所有任务统一成“根据前文补全后文”。
- 适合通用文本生成、对话、代码补全。

### 为什么主流大模型多采用 Decoder-Only

- 训练目标统一且简单，适合大规模扩展。
- 所有 token 都能直接贡献训练信号，训练效率高。
- 很适合 in-context learning，把翻译、问答、摘要都统一为补全问题。
- 业界在该架构上已经验证了明确的 scaling law 路径。

## 3. 为什么不能直接用字符或单词作为输入单元，BPE 解决了什么问题

### 直接用字符的问题

- 序列会非常长，训练和推理成本高。
- 单个字符语义太弱，模型需要跨很多步才能拼出词义。
- 长文输入时上下文窗口浪费严重。

### 直接用单词的问题

- 词表会非常大，参数成本高。
- 未登录词问题严重，新词、拼写变体、专有名词容易失效。
- 不同词形变化共享不了子结构，例如 `run`、`running`、`ran`。

### BPE 解决了什么问题

BPE 的核心思想是：从字符开始，反复合并最常见的相邻符号对，逐步形成高频子词。

它在字符和整词之间取得平衡：

- 高频常见词可以保留为完整 token，提高效率。
- 低频词可以拆成多个子词，避免 OOV。
- 相近词能共享子结构，改善泛化。

例如：

```text
understanding -> under + stand + ing
unhappiness -> un + happiness
```

所以，BPE 本质上是在解决“字符太细、单词太粗”之间的粒度问题。

## 4. 本地部署开源模型与提示策略对比

> 这道题是实践题。下面给出的是一份可复现的参考流程与示例结论，实际效果会受硬件、模型版本和 prompt 细节影响。

### 本地部署一个轻量级模型

以 `Qwen3-0.6B` 为例，可以使用 `transformers` 进行加载：

```python
from transformers import AutoModelForCausalLM, AutoTokenizer

model_name = "Qwen/Qwen3-0.6B"
tokenizer = AutoTokenizer.from_pretrained(model_name, trust_remote_code=True)
model = AutoModelForCausalLM.from_pretrained(
    model_name,
    trust_remote_code=True,
    device_map="auto"
)

def generate(prompt, temperature=0.7, top_p=0.9, top_k=50, max_new_tokens=256):
    inputs = tokenizer(prompt, return_tensors="pt").to(model.device)
    outputs = model.generate(
        **inputs,
        max_new_tokens=max_new_tokens,
        temperature=temperature,
        top_p=top_p,
        top_k=top_k,
        do_sample=True
    )
    return tokenizer.decode(outputs[0][inputs.input_ids.shape[1]:], skip_special_tokens=True)
```

### 采样参数的影响

| 参数 | 低值特征 | 高值特征 |
|------|----------|----------|
| temperature | 更稳定、更保守 | 更多样、更发散 |
| top_p | 候选更集中 | 候选更宽松 |
| top_k | 只在少数高概率词里选 | 候选词范围更大 |

经验上：

- 精确信息抽取、代码生成：`temperature=0.1-0.3`
- 一般问答：`temperature=0.5-0.8`
- 创意任务：`temperature=0.8-1.2`

### 不同提示策略对比

以情感分类为例：

- Zero-shot：最省 prompt，但小模型容易格式漂移。
- Few-shot：给几个示例，通常是小模型最划算的提升方法。
- Chain-of-Thought：对复杂推理任务帮助较大，但小模型有时会因为步骤变多而更容易跑偏。

可以用一个简单结论概括：

- 小模型优先 Few-shot。
- 大模型在复杂推理任务上更适合 CoT。

### 闭源模型和开源模型的比较

| 维度 | 闭源模型 | 开源模型 |
|------|----------|----------|
| 性能上限 | 通常更强 | 追赶很快 |
| 使用门槛 | API 即用 | 需要部署和运维 |
| 成本结构 | 按 token 付费 | 前期硬件投入更高，长期边际成本更低 |
| 隐私 | 数据发往第三方 | 可本地控制 |
| 可控性 | 受服务商限制 | 版本、微调、量化都更自主 |
| 合规性 | 部分行业受限 | 更适合高合规场景 |

### 企业级客服智能体该选哪类模型

如果是企业级客服，我更倾向于：开源模型私有化部署 + RAG + 安全审查 + 转人工机制。

需要考虑的因素包括：

- 数据隐私与合规。
- 日调用量和长期成本。
- 响应延迟是否可控。
- 是否需要学习企业私有知识。
- 是否需要微调企业风格。
- 是否有人工兜底与安全审查能力。

## 5. 缓解模型幻觉的方法

### 选择一种方法说明原理和适用场景

这里选择 RAG。

RAG 的基本流程是：

1. 用户问题先转成向量。
2. 到外部知识库中检索相关文档片段。
3. 把检索结果和问题一起喂给模型。
4. 模型基于“检索到的证据”生成回答。

它能减少幻觉的核心原因是：让模型少凭记忆，多基于证据回答。

典型适用场景：

- 企业知识问答。
- 客服系统。
- 法律和医疗咨询辅助。
- 新闻、时事、学术问答。

### 其他前沿缓解方法

除了 RAG，还可以从“推理阶段”“训练阶段”“输出阶段”三个层面缓解幻觉：

- Self-Consistency：同一问题多次采样，投票选最一致答案。
- Chain-of-Verification：先回答，再生成验证问题做自查。
- RLHF/DPO：通过偏好学习让模型更倾向诚实、谨慎回答。
- Inference-Time Intervention: 在推理时干预内部激活，提升“诚实方向”。
- Attributed Generation：要求每个关键事实都标注来源。

它们的主要区别在于：

- 是否依赖外部知识库。
- 是否需要重新训练。
- 推理成本高低。
- 更适合事实性任务还是推理性任务。

## 6. 论文辅助阅读智能体的设计

### 基座模型如何选

我会优先考虑以下能力：

- 长上下文能力，最好 32K 起步，128K 更理想。
- 强推理能力，能理解方法、实验和论证链。
- 英文论文理解能力和中英双语问答能力。
- 指令跟随性和忠实性，不能随意脑补论文内容。

如果是个人或小团队，闭源强模型使用成本最低；如果是机构级产品或对隐私要求高，则更适合高质量开源模型自部署。

### 如何设计 prompt，如何解决超长论文问题

Prompt 设计的关键是利用论文天然结构，例如：

- 研究问题是什么。
- 核心贡献是什么。
- 方法怎么做。
- 关键实验结果是什么。
- 有哪些局限。

同时要加硬约束：

- 只能基于原文回答。
- 论文没写的要明确说“原文未提及”。
- 引用结论时标注章节或段落来源。

面对超长论文，可以采用三种常见方案：

- Map-Reduce: 分章节总结后再汇总。
- 分层问答：先粗读摘要、引言、结论，再按问题精读相关段落。
- RAG: 把论文切段建索引，问题来了只检索最相关片段。

在真实产品里，我最推荐 RAG，因为它更节省上下文窗口，也更利于溯源。

### 如何确保输出准确、客观、忠于原文

系统中建议加入以下设计：

1. 引用溯源：每个关键事实都标出章节、段落或表格来源。
2. 忠实性 prompt：明确禁止脑补和越权推断。
3. 双重验证：关键数据和实验结果回到原文二次确认。
4. 幻觉检测器：自动检查回答中的事实是否被原文支持。
5. 多论文防混淆标识：对比阅读时给每篇论文分配唯一标识，避免张冠李戴。

从工程角度看，学术阅读智能体最重要的不是“说得像专家”，而是“说的每一句关键话都能在原文里找到依据”。
