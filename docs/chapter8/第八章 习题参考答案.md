# 第八章 习题参考答案

> 参考答案仅供参考。本章真正需要掌握的是：记忆系统和 RAG 不是孤立模块，而是智能体构建长期能力、外部知识能力和个体化能力的基础设施。

## 1. 四类记忆的协作与工作记忆整合

### 为什么情景记忆更强调时间近因性，语义记忆更强调图检索

情景记忆记录的是“发生过什么、在什么时间、与哪次任务有关”。它天然和时间序列强相关，因此更强调时间近因性是合理的，因为最近发生的事件往往对当前任务更相关。

语义记忆关注的是稳定知识结构，例如“某种药物属于哪类”“某个概念和哪些概念有关”。这类知识的价值不主要取决于时间，而更取决于关系网络是否准确可达，所以图检索权重更高。

可简化理解为：

- 情景记忆回答“最近发生了什么”。
- 语义记忆回答“这个知识和其他知识怎么连起来”。

### 个人健康管理助手如何组合四种记忆

| 记忆类型 | 适合存什么 | 具体场景 |
|----------|------------|----------|
| 工作记忆 | 当次会话中的即时目标和临时状态 | 今天早餐、当前步数、刚刚输入的不适症状 |
| 情景记忆 | 某次具体事件和过程 | 上周三跑步后心率过高、昨晚失眠后第二天疲劳 |
| 语义记忆 | 长期稳定知识和用户健康画像 | 用户长期对乳糖不耐受、医生建议避免高盐 |
| 感知记忆 | 来自传感器或原始观测 | 手环心率曲线、睡眠监测、体重秤数据 |

一个实际组合方式是：感知记忆采集原始数据，工作记忆支撑当前对话，情景记忆记录关键健康事件，语义记忆沉淀长期健康规则和用户画像。

### 什么时候把工作记忆整合为长期记忆

并不是所有短期信息都值得长期保存。更合适的自动整合触发条件可以是：

- 高频重复：同类偏好或约束在多次会话中反复出现。
- 高重要度：涉及风险、禁忌、过敏、慢病等关键事实。
- 跨任务价值：未来多个任务都可能用到。
- 强显式确认：用户明确说“记住这个偏好”。

一个实用评分公式可以是：

```text
consolidate_score = 0.35 * importance + 0.25 * repeat_count + 0.20 * cross_task_value + 0.20 * user_confirmation
```

当分数超过阈值时，从工作记忆升级到情景记忆或语义记忆。

## 2. RAG 分块、检索策略与嵌入模型选型

### 没有标题结构时，如何做语义边界分块

对小说、法律条文、会议纪要这类文档，不能只依赖 Markdown 标题。更稳妥的方法是“句级切分 + 语义相似度边界检测”。

参考算法：

1. 先按句子或自然段初步切分。
2. 为每个句子计算 embedding。
3. 计算相邻句段的语义相似度。
4. 当相似度显著下降，或主题发生跳转时，在此处分块。
5. 再结合最大 token 数做合并和重叠。

伪代码：

```text
for segment in segments:
    if similarity(prev, current) < threshold or token_limit_reached:
        start_new_chunk()
    else:
        append_to_chunk()
```

这样做的核心是让块边界更贴近“话题切换点”，而不是机械按长度切。

### 基础检索、MQE 与 HyDE 的效果差异

以“技术文档问答”为例：

- 基础检索：快、成本低，但容易受用户表述影响。
- MQE：把一个问题扩展成多个检索视角，适合术语多样、表达方式复杂的场景。
- HyDE：先让模型生成一段“假设答案”，再拿这段文本去检索，适合用户问题很抽象、语义稀疏的情况。

选型建议：

- FAQ 式明确问题：基础检索通常足够。
- 技术术语不统一：优先 MQE。
- 用户只会描述目标、不知道专业词：HyDE 往往更有效。

### 三种嵌入方案对比

| 方案 | 准确性 | 速度 | 成本 | 离线部署 | 适合场景 |
|------|--------|------|------|----------|----------|
| 百炼 API | 通常较高 | 取决于网络 | 按调用计费 | 否 | 快速接入、云服务优先 |
| 本地 Transformer | 高，可控 | 依机器配置 | 前期部署成本高 | 是 | 私有化、稳定长期使用 |
| TF-IDF | 语义能力弱 | 快 | 很低 | 是 | 资源紧张、关键词检索 |

结论：

- 追求最快落地可先用 API。
- 追求私有化和长期可控可选本地 Transformer。
- TF-IDF 更适合作为轻量召回或兜底方案，而不是唯一语义检索方案。

## 3. 智能遗忘、记忆归档与敏感信息彻底删除

### 设计“智能遗忘”策略

可综合重要性、访问频率、时间衰减、最近访问时间：

```text
retain_score = 0.40 * importance + 0.25 * access_frequency + 0.20 * recency + 0.15 * cross_reference_count
```

再配合一个遗忘分数：

```text
forget_score = 1 - retain_score
```

当容量超过阈值时，优先删除高 `forget_score` 的记忆。

这比单纯按时间删除更贴近真实需求，因为“重要但旧”的记忆不应轻易消失。

### 记忆归档机制怎么做

建议把长期不用但可能有价值的记忆迁移到冷存储层，例如对象存储或归档数据库。

集成方式：

- 工作记忆通常不归档，直接过期清理。
- 情景记忆可在长期不访问后转冷存储。
- 语义记忆应优先保留高价值知识，低频边缘知识可归档。
- 感知记忆可保留聚合摘要，原始大体量数据进冷存储。

检索时先查热数据；未命中再查归档索引，并按需恢复。

### 删除敏感信息为什么不能只删数据库记录

仅删主表通常不够，因为敏感信息可能还存在：

- 向量数据库 embedding
- 图数据库节点和边
- 检索缓存
- 备份和日志
- 下游生成摘要或衍生记忆

更完整的删除流程应包括：

1. 根据数据 ID 找到主记录、embedding、图节点、缓存键。
2. 级联删除所有索引和关联关系。
3. 重建受影响的摘要或派生知识。
4. 对备份设置可追踪的删除或过期策略。
5. 记录审计日志，证明删除已完成。

## 4. 学习助手中的 RAG 与 Memory 协同

### 什么时候优先用 RAG，什么时候优先用 Memory

优先用 RAG：

- 问题依赖外部知识库内容。
- 需要查教材、讲义、题库、文档原文。
- 用户问题更像“知识查询”。

优先用 Memory：

- 问题依赖用户个人学习历史。
- 需要结合已错题目、偏好、进度、长期薄弱点。
- 用户问题更像“个体化辅导”。

### 智能路由机制设计

可以先做一个轻量路由器，判断问题属于哪一类：

- `knowledge_query`
- `personal_progress_query`
- `hybrid_query`

路由特征可包括：

- 是否包含“我之前”“我上次”“我的进度”等个人化表达
- 是否涉及教材章节、定义、公式等知识库表达
- 是否需要同时参考用户历史和外部知识

混合问题则采用“双路检索 + 重排序融合”。

### 更智能的学习报告生成器

一个更好的报告不应只有统计计数，还应包含：

- 学习轨迹：最近一段时间学了什么、在哪些主题停滞。
- 知识盲点：高频错误概念、反复遗漏步骤。
- 下一步建议：先补什么，再练什么。

需要用到：

- 情景记忆：记录每次学习事件。
- 语义记忆：建立知识点之间的关联。
- RAG：从教材中找对应补充材料。
- 混合检索：把“用户弱项”映射回“相关知识内容”。

### 多用户 Web 服务的数据隔离

Qdrant 中可通过 `payload.user_id`、独立 collection 或分片路由做隔离；Neo4j 中可通过 `tenant_id` 标签、子图隔离或独立数据库做隔离。

优化建议：

- 高频用户走热缓存。
- 按用户或租户做分区，减少无关扫描。
- 对检索先做用户级过滤，再做向量相似度计算。

核心原则是：隔离应优先在索引层和查询层完成，而不是检索后再过滤。

## 5. 知识图谱质量与图检索优势

### 自动抽取实体关系的准确性问题

自动抽取通常能显著节省人工成本，但准确性并不稳定，常见错误包括：

- 实体边界错误，例如把“高血压患者”误抽成“高血压”。
- 关系方向错误，例如把“药物治疗疾病”反过来。
- 上下文歧义导致错连，例如同名概念跨领域混淆。
- 幻觉关系，即文本没明确说，模型却自行补出关系。

### 知识图谱质量评估机制

可以从四个维度做评估：

- 抽取准确率：抽样人工校验实体和关系。
- 图结构一致性：检查孤立节点、非法关系、环异常。
- 来源可追溯性：每条边都能回溯原始文本证据。
- 下游任务收益：看图检索是否真正提升问答效果。

### 一个纯向量难以完成的图查询场景

例子：查询“某门课程的先修知识链路，并找出该学生当前缺失但又会影响下一阶段学习的关键节点”。

这需要：

- 多跳关系推理
- 路径查找
- 将学生已掌握知识与课程知识图谱比对

纯向量检索可能找出相关文本，但很难稳定给出“依赖路径”。图查询则更自然。

### 图检索在哪些查询中明显优于纯向量检索

图检索在以下问题中优势明显：

- 多跳关系问题：A 影响 B，B 影响 C，问 A 如何影响 C。
- 路径依赖问题：完成任务 X 前必须掌握哪些前置知识。
- 实体间结构关系问题：某疾病、药物、症状、禁忌之间的联动。

纯向量更擅长语义相似，图检索更擅长结构推理。两者结合通常优于单独使用。
